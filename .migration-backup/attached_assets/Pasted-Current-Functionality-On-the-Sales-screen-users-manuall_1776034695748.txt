Current Functionality

On the Sales screen, users manually update closing stock using the following fields:

Cls Bal (Cs)
Cls Bal (Btls)
Breakage

Based on these inputs, the system automatically calculates:

Sold Bottles
Total Closing Stock
Final Closing Stock Balance
Sold Stock Value (displayed at the top)
Closing Balance Value (displayed at the top)
Bottle-level calculations

Currently, users must manually enter closing stock data for each Brand No and Size on the Sales screen.

Enhancement Requirement: Excel-Based Bulk Update
📥 1. Excel Template Download

Provide a downloadable Excel template for bulk data entry.

✅ Template Structure

The template should include only the following fields:

Brand No
Brand Name
Size
Cls Bal (Cs)
Cls Bal (Btls)
Breakage (if applicable)
🔁 Template Data Generation (Important)

The template should not be blank.

Instead:

It should be pre-populated with existing data from the current Sales screen.
Specifically, include:
All Brand No + Brand Name + Size combinations currently displayed for that page/context.

👉 In other words:

The template should behave similar to the existing “Current Export CSV” feature
But with the following differences:
Include only required columns (not all fields)
Keep Brand/Size data prefilled
Leave editable fields (Cls Bal (Cs), Cls Bal (Btls), Breakage) for user input/update
📤 2. User Input
User downloads the template
Updates only:
Cls Bal (Cs)
Cls Bal (Btls)
Breakage (if needed)
⚙️ 3. Upload & Processing
User uploads the completed Excel file
System processes each row and:
Matches records using:
Brand No
Size (and/or Brand Name if needed)
Applies the same calculation logic as the current system
🧮 4. Calculations (Same as Existing Logic)

For each uploaded record, system should automatically calculate:

Sold Bottles
Total Closing Stock
Final Closing Stock Balance
Sold Stock Value
Closing Balance Value

IMP : Must be identical to current Sales screen calculations

Key Clarification:

This feature is essentially:

A filtered and simplified version of “Current Export CSV”
➝ Used as an input template instead of just export

If user not enter any Value that means there is no sale that day

⚠️ Validation & Handling (Recommended)
Validate:
Missing Brand No / Size
Invalid numeric values
Handle:
Unknown brands/sizes (reject or skip)
Duplicate rows
Provide:
Upload result summary (success / failed rows)
